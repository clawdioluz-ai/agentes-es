# Backlog de decisões arquiteturais

## Objetivo
Registrar as decisões arquiteturalmente significativas que a próxima fase precisa fechar, sustentar ou refinar.

## Decisões já fortemente encaminhadas
- postura control-plane-first
- workflow backbone durável
- governança human-in-the-loop
- runtimes especialistas pluggable
- evidência e auditabilidade como preocupações centrais
- UI tratada como mission control

## Decisões ainda abertas
- granularidade do modelo canônico de objetos
- fronteira entre control plane e runtime cognitivo
- forma de encapsular políticas e alçadas
- estratégia de memória por tipo e por retenção
- desenho do gateway de integrações
- pacote mínimo de visões e documentação para a próxima fase

## Para cada decisão
- contexto
- opções
- critérios
- tradeoffs
- implicações
- dependências
- recomendação provisória
