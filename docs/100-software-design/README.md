# Software design

## Objetivo
Abrir uma seção dedicada ao desenho conceitual da plataforma de orquestração, transformando a pesquisa já consolidada do repositório em artefatos de design mais diretamente utilizáveis na próxima fase.

## Escopo
Esta seção continua deliberadamente:
- conceitual
- orientada a arquitetura e produto
- sem implementação
- sem protótipos
- sem detalhamento técnico executável

## Pergunta central
Quais artefatos de design devem existir para traduzir a tese control-plane-first deste repositório em um pacote coerente de arquitetura de software para a próxima fase?

## Estrutura inicial
- `01-design-artifact-stack.md`: pilha recomendada de artefatos de design
- `02-system-context-and-boundaries.md`: contexto do sistema, fronteiras e relações externas
- `03-views-and-viewpoints.md`: conjunto mínimo de visões arquiteturais e seus públicos
- `04-quality-attribute-scenarios.md`: cenários de atributos de qualidade para orientar decisões
- `05-conceptual-domain-model.md`: modelo de domínio conceitual para a plataforma
- `06-decision-backlog.md`: backlog inicial de decisões arquiteturalmente significativas

## Relação com o restante do repositório
Esta seção não substitui `docs/40-orchestration/` nem `docs/90-adrs/`. Ela funciona como ponte entre:
1. pesquisa e síntese estratégica
2. proposta conceitual de plataforma
3. pacote de artefatos de design que prepara a fase seguinte
