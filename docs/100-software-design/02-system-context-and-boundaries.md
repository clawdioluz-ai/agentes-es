# Contexto do sistema e fronteiras

## Objetivo
Delimitar o sistema que esta pesquisa propõe, diferenciando núcleo de plataforma, superfícies, integrações e elementos externos.

## Escopo inicial
### Dentro do núcleo conceitual
- control plane
- workflow backbone
- runtime de especialistas
- memória governada
- policy and approval fabric
- observabilidade e evidência
- mission control UI

### Fora do núcleo, mas conectado
- git e plataformas de código
- CI/CD
- documentação
- backlog e issue tracking
- sistemas de identidade
- provedores de modelos

## Perguntas de design
- onde termina a plataforma e começam os sistemas do SDLC?
- o que deve ser tratado como capacidade nativa versus integração?
- quais fronteiras precisam permanecer pluggable?
