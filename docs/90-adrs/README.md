# ADRs iniciais da plataforma de orquestração

## Objetivo
Registrar, em formato curto e estável, as decisões arquiteturais e de produto mais estruturantes já implicadas pela pesquisa deste repositório.

## Escopo
Estas ADRs são deliberadamente:
- conceituais
- estratégicas
- orientadas a arquitetura e produto
- não implementacionais

Elas não escolhem APIs, schemas técnicos, frameworks finais, nem detalhes de deploy. Seu papel é fixar a direção para a próxima fase.

1. Ler `ADR-001` e `ADR-002` para entender a espinha dorsal da plataforma.
2. Ler `ADR-003` e `ADR-006` para entender governança, risco e evidência.
3. Ler `ADR-004` e `ADR-005` para entender a postura de composição e extensibilidade.
4. Ler `ADR-007` para entender a implicação de produto e interface.
